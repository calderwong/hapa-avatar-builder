# Overcard navigation and telemetry

Every Builder attachment inspector links to its source entity, Responsibility Binding when present, owning process, and up to three recent safe evidence envelopes. Cross-node resources use `hapa://overcard/<node>/<source|binding|process|evidence>/<id>` with optional revision and surface query parameters; local resolvers and capability endpoints remain ordinary relative links.

Evidence links expose only id, summary, time, level, and deep link. They do not expose prompts, raw input/output, memory, credentials, or grants.

Builder mounts the package-owned telemetry badge. Dev Proto projects that same truth through its single system status bar, without a duplicate badge. The shared states are `healthy`, `healthy-zero`, `absent`, `stale`, `degraded`, and `offline`. “Not reported” is never displayed as zero activity, and a stale projection is never displayed as healthy. Both consumers name the same shared host revision.
