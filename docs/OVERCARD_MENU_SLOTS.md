# Overcard menu slots

Every generated Builder tab includes a diamond attachment hitbox and count. Opening it mounts the canonical `OvercardHostSlots` component with that registry target's typed socket. A held entity can be staged by click or canonical MIME drop; native file drops are not intercepted.

The panel states view/process/responsibility mode, effect, compatibility, capacity, lifecycle, health, effective context, and fallback in text. It supports stack inspection, earlier/later reorder, context activation or authority request, disable, replacement with displaced-card return, Detach + return, and Undo. Full, incompatible, unauthorized, stale/conflict, offline, and degraded results remain visible and suggest a recovery action.

Placement and attachment records commit through the shared revisioned host—not local navigation state—so they survive view changes, restarts, and appear in the other app's subscription. Menu popover/open state is renderer-local and intentionally does not sync.
