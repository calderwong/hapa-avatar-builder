# Echo reference graph → Avatar Genesis update

## Objective

Make Avatar Genesis consume the Echo reference graph and current song store while preserving the operator-confirmed fact that `Echo Album` is a later music-visualizer projection of the substantially same `Dear Papa` songs.

## Evidence inspected

- `data/hapa-songs-store.json`: 109 songs, 52 reference entries, 30 graph edges, 32 connector-bearing songs.
- Existing Avatar Mind Song Choices: 1,214 selections across 74 Avatars; all resolve to the current song store.
- `docs/ECHO_REFERENCE_GRAPH.md`, the Genesis prompt/schema, Avatar Mind normalization, recovered-roster Genesis, Tarot enrichment, card-song kit enrichment, and Ballad of Bella lore pass.

## Reused

- Existing Echo connector/context/graph schemas.
- Existing Avatar Mind `dearPapaSongContext` compatibility field.
- Ballad of Bella's pattern of converting sourced song semantics into persona, self-knowledge, context, memory, lexicon, and Genesis receipts.
- Existing source/canon/review boundaries.

## Changed

- Added explicit Dear Papa/Echo album lineage and stable identity rules.
- Added a current-song/reference-graph gate to the canonical Genesis prompt and output schema.
- Extended Avatar Mind Song Choices to preserve connector snapshots, contextual layers, reference summaries, semantic primitives, graph edge IDs, and graph hash.
- Added a repeatable dry-run/apply update for existing Avatars with backup and report behavior.

## Noise or misunderstanding corrected

The earlier audit treated the missing Echo→Mind bridge as if both systems had been designed concurrently. The Echo graph was added after the original Genesis implementation. The gap is chronological, not evidence that the original Genesis deliberately rejected semantic ingestion.

`Echo Album` is not a separate replacement corpus simply because the visualizer uses a later album title. Album title is projection metadata; song/card IDs and lyric hashes carry continuity.

## Guardrails

- Never delete or collapse prior Song Choices during graph hydration.
- Never infer hard Avatar biography from an external reference or personal association.
- Preserve literal evidence, public reference, personal context, and comparative inference as distinguishable layers.
- A future graph update appends or reweights interpretation; it does not rewrite lyric source text.
